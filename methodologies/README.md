# Methodologies

Methodologies are lazy packs. Keep Kanban available by default; initialize
others only on first use.

| Pack | Status | When used |
|---|---|---|
| `kanban.md` | default | normal PM flow |
| `tdd.lazy.md` | lazy | when owner asks for TDD or risk calls for test-first |
| `code-review.md` | available | code review or PR review |
| `adr.md` | available | non-trivial decisions |

If a methodology has stack-specific conventions, ask before searching online.
Prefer official docs and primary sources.
