# ADR Methodology

Use ADRs for decisions that change project direction, stack policy, architecture,
team governance, security posture, or documentation obligations.

## ADR Shape

```md
# ADR NNNN: Title

Date: YYYY-MM-DD
Status: Proposed | Accepted | Superseded | Deprecated

## Context

## Decision

## Considered Alternatives

1. Option A - trade-off.
2. Option B - trade-off.

## Consequences

## Follow-ups
```

Keep alternatives capped at two unless the owner asks for broader exploration.
